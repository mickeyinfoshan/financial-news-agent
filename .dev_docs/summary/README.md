# .dev_docs/summary

Development summaries and retrospectives.

## Purpose

Post-implementation documentation capturing what was built and lessons learned.

## Ownership

Retrospective documentation written after features are implemented.

## Contents

- `finnhub-integration.md` - Finnhub API integration summary
- `dynamic-ticker-lookup.md` - Dynamic ticker lookup implementation notes
- `documentation-update.md` - Documentation update summary
- `context-window-management-implementation.md` - Context management implementation summary
- `citation-system-implementation.md` - Numbered citation system implementation
- `retry-mechanism.md` - Retry/fix mechanism for automatic quality improvement
- `retry-mechanism-changes.md` - Updates and improvements to retry mechanism

## What Goes Here

- Implementation summaries and retrospectives
- Lessons learned from completed work
- Notes on how features were built
- Technical debt and future improvement notes

## What Doesn't Go Here

- Planning documents (use `.dev_docs/plan/` instead)
- User-facing documentation (use `docs/` instead)
- Temporary test scripts (use `.dev_process/` instead)

## Reference

See `.claude/rules/documentation-organization.md` for the full documentation organization policy.

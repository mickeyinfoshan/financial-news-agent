# .dev_docs/plan

Implementation plans and design documents.

## Purpose

Planning phase artifacts created before implementation begins.

## Ownership

Design documents, implementation plans, and architectural decisions.

## What Goes Here

- Implementation plans for new features
- Design documents for architectural changes
- Technical decision records
- Planning documents with dates when relevant (e.g., `2026-05-23-feature-name.md`)

## What Doesn't Go Here

- Post-implementation summaries (use `.dev_docs/summary/` instead)
- Temporary test scripts (use `.dev_process/` instead)
- User-facing documentation (use `docs/` instead)

## Reference

See `.claude/rules/documentation-organization.md` for the full documentation organization policy.
